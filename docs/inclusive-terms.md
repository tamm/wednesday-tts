# Inclusive Terminology

Use modern, inclusive terms. The old ones are outdated and exclusionary.

## Quick Reference

| Avoid           | Prefer                                         |
| --------------- | ---------------------------------------------- |
| whitelist       | allowlist                                      |
| blacklist       | blocklist, denylist, excludelist               |
| master (branch) | main                                           |
| master/slave    | primary/replica, leader/follower, parent/child |
| sanity check    | validity check, confidence check               |
| dummy value     | placeholder, sample value                      |
| grandfathered   | legacy, exempted                               |

## Guidelines

- Be specific where possible: "excludelist" if excluding, "denylist" if denying access
- When in doubt, describe the function rather than using jargon
- The default branch is `main`, not `master`
- This applies to all code, docs, comments, and commit messages in this repo
